# Clean code

## Definitions
- "Clean code is simple and direct"
    - Grady Booch (Creator of UML)

- "The logic should be straightforward and make it hard for bugs to hide"
    - Bjarne Stroustrup (Creator of C++)

- "Clean code always look like it was written by someone who cares"
    - Michael Feathers (Author of Working Effectively with Legacy Code)

- "You know you are working on clean code when each routine you read turns out
  to be pretty much what you expected"
    - Ward Cunningham (Creator of Wiki)

- "Any fool can write code that a computer can understand. Good programmers write code that humans can understand"
    - Martin Fowler (Author of many great books about software development)

- "The only valid measurement of code quality: WTFs/minute"
