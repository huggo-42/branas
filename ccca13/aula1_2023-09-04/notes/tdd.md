- the cicle
# RED
1. Write a test that fails
# GREEN
2. Make the code work
# REFACTOR
3. Eliminate redundancy

---

# Three laws of TDD
1. You cannot write any code if you haven't written a test that detects
   possible failures
2. You cannot write more unit tests then the suficient to detect a failure
3. You cannot write more source code then the needed to pass through all the tests
    - Robert C. Martin
