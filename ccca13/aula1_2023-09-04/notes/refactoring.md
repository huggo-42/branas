# Refactoring

- "A change made to the internal structure of software to make it easier to
  understand and cheaper to modify without changing its observable behavior."
    - Martin Fowler

## Why
- Refactoring is an investment, makes the software sustainable and competitive
- Daily, not refactoring takes more and more time of the team

## When
- to refactor should not be in backlog
- Refactor when:
    - adding new features
    - fixing a bug
    - when you need to understand a branch of code
